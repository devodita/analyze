Automated Data Processing & Reporting

Leveraging Python, Pandas, and GitHub Actions for efficient data workflows.

Project Overview

This project demonstrates a robust data processing pipeline using Python and Pandas, orchestrated by GitHub Actions. It takes raw data from an Excel file (data.xlsx), processes it with a Python script (execute.py), and publishes the structured output (result.json) via GitHub Pages.

The pipeline also includes code quality checks using Ruff and ensures data integrity through careful error handling in the processing script.

Workflow Components

1. Data Input

Raw data is provided in an Excel spreadsheet: data.xlsx. A CSV version (data.csv) can also be generated from it.

2. Processing Script

The execute.py script, written in Python with Pandas, reads, cleans, aggregates, and transforms the data. It's designed with robust error handling.

3. CI/CD Pipeline

A GitHub Actions workflow (.github/workflows/ci.yml) automates linting (Ruff), execution of the script, and deployment.

4. Output & Deployment

The processed data is output as result.json, which is then published as a static site via GitHub Pages, making it easily accessible.

Accessing the Output

The final processed data (result.json) is generated dynamically by the CI/CD pipeline and deployed to GitHub Pages. You can typically find it at a URL similar to: https://<your-username>.github.io/<your-repository>/result.json.
